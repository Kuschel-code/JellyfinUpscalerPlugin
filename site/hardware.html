Hardware guide

Which GPU + how much VRAM + which execution provider you actually need. No marketing fluff.

Client-side VSR vs. server-side upscaling - which one?

There are two fundamentally different ways to make a stream look sharper, and they are not interchangeable. Pick the wrong one and you either melt a Celeron NAS or get shader-grade results when you wanted Real-ESRGAN. This plugin is the server-side path; client-side VSR is the alternative you should reach for on weak hardware.

Server-side AI plugin (this project)Client-side VSR (Anime4K / driver VSR)
Runs onThe Jellyfin server's GPU/CPUThe playback device's GPU (your TV box, phone, desktop)
EngineDeep-learning SR (Real-ESRGAN, SwinIR, HAT, …)Real-time shaders (Anime4K) or vendor VSR (NVIDIA RTX VSR, Intel, AMD)
Quality ceilingHighest - true learned detail reconstructionGood for anime/line-art; shader-grade, not learned
Server costHigh (GPU/CPU per stream)Zero - the server just sends the original file
Result is cachedYes - upscale once, replay freeNo - recomputed on every device, every play
Works on a weak serverOnly light models (fsrcnn-x2), batch/overnightYes - server load is unchanged
Needs a capable clientNo - any client plays the re-encoded streamYes - modern GPU + WebGL/MPV-capable player
Rule of thumb: strong server GPU → server-side (best quality, cached). Weak server (Celeron/N100/NAS) + good playback device → client-side VSR. This is exactly what the built-in /recommend endpoint encodes: on a weak-CPU box it points you at Anime4K on the client GPU instead of pushing Real-ESRGAN onto the server.

Anime4K (built in)

The plugin ships an embedded Anime4K tier - a WebGL shader that runs in the browser/player on the client GPU, zero server cost. Ideal for anime libraries served from a low-power box. Enable it in Config → Real-time; it has no effect on clients without a capable GPU and silently falls back to the original stream.

Vendor VSR (RTX VSR / Intel / AMD) - client-only by design

NVIDIA RTX Video Super Resolution, DLSS, AMD FSR and Intel's equivalents are driver/GPU features of the viewing device. The server cannot invoke them - there is no API to drive another machine's display pipeline. If your client supports RTX VSR, turn it on in the GeForce/driver settings; it composes fine with a server that just streams the source untouched.

The four providers, ranked

TierProviderWhat it looks like
1NVIDIA CUDA + TensorRTRTX 2060 and up, ≥6 GB VRAM. Best latency, widest model support, TensorRT FP16 paths on Ampere+.

Recommended setups

Budget home lab (Intel N100 / Arc A380)

  • Provider: OpenVINO (:latest-openvino image)
  • Good for: FSRCNN real-time, Waifu2x 2×, Real-ESRGAN-anime-x4 batch
  • Config: Max Concurrent Streams = 1, cache enabled
  • Expect: ~15 fps at 720p → 1440p with anime-compact-x4

Mid-range (RTX 3060 12GB / RTX 4060)

  • Provider: CUDA (:latest-cuda image)
  • Good for: Everything in the catalogue except EDVR-M and HAT-L comfortably
  • Config: Max Concurrent Streams = 1, Auto-Mode on
  • Expect: 4×-realesrgan at ~20 fps on 720p source, 3× on 1080p source

High-end (RTX 4090 / A100)

  • Provider: CUDA + TensorRT conversion
  • Good for: SwinIR-L, HAT-L, EDVR-M temporal, GFPGAN pipeline
  • Config: Max Concurrent Streams = 2, pre-processing cache on NVMe
  • Expect: Near real-time 4× on any model

CPU-only NAS (e.g. Synology DS920+)

  • Provider: CPU (:latest-cpu image)
  • Good for: Overnight batch upscales of an anime library with fsrcnn-x2/espcn-x4
  • Config: Scan & Upscale Library scheduled at 3 AM, pre-processing cache on the biggest pool you have
  • Expect: Hours per feature-length file. This is batch, not live.

Jellyfin-side hardware acceleration

Orthogonal to the AI upscaler - Jellyfin still handles decoding the source file and encoding the final MP4 itself.

  • Decoders: NVDEC, QSV, VAAPI all work unchanged.
  • Encoders: the plugin exposes 12 codec options with tuned defaults (see Config → Output Codec).
  • Tonemap: HDR→SDR via tonemap_cuda / tonemap_opencl handled by Jellyfin's stream path, runs before the upscale filter.
