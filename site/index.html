What makes it different

Docker microservice architecture

The AI runtime runs in its own container, isolated from Jellyfin's .NET process. No native DLL conflicts, no GPU driver coupling, update either side independently.

Real model catalog

Not a wrapper around one model — 30+ ONNX upscalers across Real-ESRGAN / SwinIR / HAT / EDVR / AnimeSR / RIFE families, with live status + hot-reloadable loader.

Live in-player controls

Tabbed quick-menu over the Jellyfin video player: Models, Filters (15 presets + 3 live CSS sliders), Realtime. No re-transcode for look tweaks.

Hardware-aware routing

Auto-benchmark on first run, pick the best provider (CUDA → DirectML → OpenVINO → ROCm → CPU), log per-model throughput, route jobs accordingly.

Inspectable REST API

Both the plugin and the AI service expose OpenAPI — browse endpoints live, call them from the console, or wire them into your own automation.

Self-hosted & private

No SaaS calls, no telemetry. Optional shared-secret API token between plugin and AI service. Runs fully offline once models are downloaded.

Latest release

v1.7.0 · Anime4K Realtime + Adoption Completion + Settings-Import Fix

Substantial release: NEW Anime4K realtime tier (real AI-shader upscaling for anime via the MIT-licensed Anime4K.js library, loaded from jsdelivr CDN with Lanczos fallback). The existing "WebGL" mode is honestly rebranded to "Lanczos + Sharpen" (it was always classical, never AI). Plus all P0/P1 audit findings closed in one go: Frame-Loop cancellation token propagation (closes the v20/v21 outer-loop adoption hole), WaitForExitAsync(ct) with linked CTS, debounced async PersistQueue, 5 missing HttpContext.RequestAborted, 18 missing TryApply lambdas for Settings-Import (filter properties, face-restore, library-IDs, AI-service token), and new QualityLevelRegistry + ButtonPositionRegistry with HTML-parse drift-locks (same pattern as v23's CodecRegistry). +13 regression tests, 102 → 115 passing. Build 0/0. RealtimeMode dropdown: Auto / Lanczos+Sharpen / Anime4K / Server-AI. WebGPU+ONNX-Runtime-Web Real-ESRGAN realtime path is roadmap for v1.7.1 (separate test cycle). Saved v1.6.x configs are bit-for-bit compatible.

Hotfix for a high-impact silent bug found in the v22 deep-audit. The Settings #OutputCodec dropdown advertised 12 codec options across 4 optgroups (Software / NVIDIA / Intel / Stream-Copy), but four code paths each had their own inline allowlist of different sizes — most critically the UpscalerController save endpoint accepted only 3 (libx264, libx265, copy). Picking AV1, NVENC, or QSV in the dropdown silently fell back to libx264 on save. On NVIDIA hardware that's a 5-20× encoding-speed regression the user couldn't see (no error, no toast). v23 introduces Services/CodecRegistry.cs as single source of truth: OutputCodecs (all 12) for save/reconstruct/batch and RealtimeOutputCodecs (HW-encoders + libx264/265, no copy/software-AV1) for the realtime pipe path. All four sites now reference one of these sets — no more drift. New CodecRegistryTests (+17 tests) parses the embedded HTML and asserts set-equality against CodecRegistry.OutputCodecs at build time, so adding a UI option without updating the registry (or vice versa) fails tests. dotnet test 85 → 102 passing. dotnet build 0/0 errors. Zero deletions, only consolidation: existing inline allowlists replaced with CodecRegistry.* references.

Who it's for

  • Home-lab operators running Jellyfin on TrueNAS SCALE, Unraid, Synology, or bare Docker who want a real upscaler they control.
  • Anime / old-TV libraries where 480p is dominant and bicubic looks terrible — AnimeSR + camera filters deliver visible gains.
  • Mixed GPU environments where you want the same plugin to work across NVIDIA desktops, Intel N100 mini-PCs, and CPU-only NAS boxes.
  • Developers who want to self-host an ONNX inference service behind an auth-gated REST API and script their own pipelines.

Quick start

Two steps — add the plugin, run the AI service.

# 1. Jellyfin: add repository URL under Dashboard → Plugins → Repositories
https://raw.githubusercontent.com/Kuschel-code/JellyfinUpscalerPlugin/main/repository-jellyfin.json

# 2. Host: run the Docker AI service (CUDA build shown)
docker run -d --name jellyfin-ai-upscaler \
  --gpus all -p 5000:5000 \
  -v upscaler-models:/app/models \
  kuscheltier/jellyfin-ai-upscaler:latest-cuda

# 3. Plugin settings → AI Service URL: http://<host>:5000 → Test Connection
